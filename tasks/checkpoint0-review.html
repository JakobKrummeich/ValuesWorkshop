DDD depth — the review focus of this gate

  • Ubiquitous language: de-first glossary; every SPEC noun mapped or explicitly excluded; zero technical vocabulary in any Phase-0 doc.
  • Aggregates as consistency boundaries: Session root guards + routes only; 7 internal building blocks each own one state slice + its invariants (anti-god-class guardrail written into the doc). Group is a separate aggregate so groups edit without contention.
  • Read models, not aggregates: answer/selection/vote tallies + presentation view are recomputable projections enforcing nothing.
  • Events past-tense, commands imperative, each command names exactly one actor (Facilitator / Participant / Scribe / System).
  • Invariant ownership: each of I1–I16 enforced by exactly one aggregate — the only place that can refuse the violating command.

Locked decisions carried into implementation

  • No join form: QR → sign-in → lobby; JoinSession implicit.
  • Facilitator quiz control = one morphing button (Reveal → Learning text → Next question).
  • Presenter groups: paginated 3×2 cards, 7 s cycle (phases 5+6).
  • Group work: value tabs; member view read-only, 0.5 s sync.
  • Presenter during voting: “Voting ongoing…”, no tallies.
  • Phase 9: one full screen per winner (with actions), least → most voted; participant PDF button only after conclusion.
  • Facilitator may typo-fix presented actions in phase 7 (T17a) — wording only.
