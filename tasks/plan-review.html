Remaining decisions

1 · Package manager: what pnpm is

pnpm is a drop-in replacement for npm — same package.json, same registry, same commands (pnpm install, pnpm run dev). It differs in how it lays out node_modules:

npmpnpm
node_modules layoutflat — all transitive deps hoisted to top levelnested via symlinks into a global content-addressed store
Phantom dependenciespossible: code can import packages you never declared (they were hoisted by some dependency) — breaks randomly on updatesimpossible: only declared deps are resolvable; undeclared import fails instantly
Monorepo workspacessupported, weaker filteringfirst-class (pnpm --filter), fits frontend + devtools/oidc
Install speed / diskcopies every package per projecthardlinks from shared store — faster CI, less disk
Maturitydefault, ubiquitousmainstream since years (Vue, Vite, Next.js repo itself use it)

Why proposed here: the phantom-dependency strictness is a deterministic gate by construction — same philosophy as the rest of the pipeline. Cost: contributors need corepack enable / npm i -g pnpm once. npm is fine too if you prefer zero extra tooling.
