§1 Ubiquitous language — key modeling decisions

  • Presenter view is not an actor — a shared observing view; it never issues commands.
  • Scribe is a role within a Group, not a separate person kind; rights move instantly on reassignment.
  • Reconnect (Wiederverbindung) — per your feedback, called what it is: come back, resume exact prior place. Delivery mechanics still excluded → protocol.md.
  • Workshop record (Workshop-Protokoll) is the domain term for the take-home output; PDF/download excluded as format concern.
  • Group sizing rule stated in prose (≥1 group, N÷4 rounded down, round-robin deal-out, sizes differ ≤1); CP-SAT math excluded → cpsat-model.md.
  • Explicitly excluded nouns table: QR code/join URL, login/identity provider, server/client/broadcast, restart/persistence, solver/time-box, PDF, charts/animations, config files — each with a pointer to where it lives instead.

§4 Aggregates

Session root

One consistency boundary — but composed of small internal building blocks (pure domain objects, no storage connotation), each owning one slice + its invariants: Roster (I4) · Workshop state (I1, I2) · Quiz progress (I5) · Selection round (I6, I7) · Formation record (I8) · Presentation walk (I12) · Voting rounds (I13–I15).

Root does two things only: guard boundary (phase + actor) and route each of its 16 commands to exactly one block. Doc now carries an explicit implementation guardrail: root implementing block rules itself = god class = violates the model; complexity gate 10 + review catch it.

Group per group

Created by GroupsFormed; membership + assigned values fixed thereafter. Owns: animal name · members · assigned values · scribe · actions · submitted state.

Handles 6 commands: ReassignScribe + all scribe editing/submit commands. Separate aggregate so groups edit concurrently without contention.

Read models (not aggregates): Answer tally, Selection tally, Vote tally (counts only — anonymity by construction), Presentation view. Enforce nothing; recomputable from events.

Resolved: aggregate = consistency/transaction boundary, not responsibility module; SRP lives one level below. Per your naming concern, the doc says “internal building blocks” (pure domain objects) — never “entities”, to avoid persistence-layer association — and carries an anti-god-class guardrail aimed at implementer agents.

Deliberate modeling calls (challenge these)

  • Quiz/selection state lives in Session, not own aggregates — tallies are session-wide, single writer path per participant action; splitting bought nothing.
  • Group is a separate aggregate — independent concurrent editing per group; scribe/action invariants (I9–I11) live wholly inside it.
  • Quiz answers are per-participant tracked (dedup) but not secret; final votes are anonymous & secret by I14. Confirmed by you — and consequence now explicit in I14/FinalVotesSubmitted: no un-voting possible.
  • System commands (DetermineTopValues, FormGroups, AppointScribes) model phase-entry effects as first-class commands with actor "System" — keeps the state machine (0.2) purely command→event.
  • WinnersDetermined fires on CloseVoting without tie — tie state is Session sub-state, not an aggregate.
